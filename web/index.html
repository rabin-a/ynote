How it works GitHub

Runs in your browser · Nothing leaves your machine

Your notes. One renderer.
Zero backend.

A markdown editor, live preview, and exporter that opens a folder on your own disk and writes plain .md files back to it. The same Rust engine powers the preview, the export, the desktop app, and the agent tools — so what you see is exactly what you ship.

See the three ways to run it

Use it here, in the browser

Click Open a folder. Pick any directory of markdown on your disk — ynote reads and writes those exact files. No upload, no account, no sync. Best in Chrome, Edge, Brave, or Arc.

Install the desktop app

Prefer a native window with system fonts and offline PDF export? The desktop build is the same engine wrapped in a 1-second-cold-start app for macOS, Windows, and Linux.

Download

Wire it into your agent (MCP)

Let Claude or any MCP client read, write, and export your notes directly. Point the server at a folder and the agent gets a small, typed tool surface — write_document, edit_section, export. 

{
  "mcpServers": {
    "ynote": { "command": "ynote-mcp", "args": ["--project", "."] }
  }
}
